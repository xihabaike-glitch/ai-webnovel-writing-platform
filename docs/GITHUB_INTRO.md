## 中文 README 开头

AI 网文写作平台不是“一键生成小说”的聊天玩具，而是一个面向作者和工作室的写作生产系统。它把作品工作台、章节、大纲树、人物弧光、世界观、伏笔、平台策略、模型岗位、任务派单、失败修复和发布包放进同一条可验收流水线里。

产品当前锁定 8 个核心平台：起点中文网、番茄小说、七猫、晋江文学城、知乎盐选、WebNovel、Royal Road、Wattpad。作者先写开头和结尾，再写主干，最后展开分支、叶片和土壤；AI 模型只作为编辑部岗位生成候选稿、审稿、二改、整理资料和包装投稿，最终采用权始终在作者手里。

## English README Intro

AI Webnovel Writing Platform is a production workspace for long-form web fiction. It is not a one-click novel generator. It helps authors manage story structure, chapters, character arcs, worldbuilding, foreshadowing, platform strategy, AI editorial tasks, dispatch receipts, failure recovery, quality gates, and submission packages.

The platform supports eight target channels: Qidian, Fanqie, Qimao, Jinjiang, Zhihu Yanxuan, WebNovel, Royal Road, and Wattpad. Claude, DeepSeek, Kimi, GPT, and OpenAI-compatible models are treated as editorial roles, not as uncontrolled text generators. AI output stays as drafts or evidence until the author accepts it.

## 项目亮点

1. 大树写作法：开头、结尾、主干、分支、叶片、土壤分层管理。
2. 8 个平台策略：覆盖国内付费、免费广告、女频、短篇和海外连载平台。
3. AI 编辑部：Claude、DeepSeek、Kimi、GPT 分别承担结构、中文写作、长上下文和海外包装岗位。
4. 候选稿机制：AI 输出不直接覆盖正文，必须经过人工采用。
5. 派单回执：每个任务都保留角色、输入、输出、人工验收和下一步。
6. 主控闸门 闸门：没有样本、复查、失败修复和发布包证据，不允许批量生产。
7. 失败修复中心：按模型配置、提示词上下文、样本重试和人工复盘拆解恢复路径。
8. 发布包与复盘：从样章、标签、卖点、版本基线到真实反馈形成闭环。

## 适合谁

1. 新人作者：需要选平台、搭结构、跑首章样本。
2. 稳定更新作者：需要管理长篇、人物、伏笔和连续性。
3. 工作室作者：需要多模型分工、任务派单和验收回执。
4. 出海作者：需要 WebNovel、Royal Road、Wattpad 的英文包装和平台适配。

## 不是什么

1. 不是纯聊天机器人。
2. 不是一键生成百万字小说。
3. 不是自动投稿工具。
4. 不是用炫技页面替代真实写作闭环的演示项目。

让 AI 像编辑部一样协作，让作者像主编一样掌控网文生产。
