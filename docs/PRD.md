# AI 网文写作平台 PRD

日期：2026-07-01
版本：MVP 0.1

## 1. 产品结论

本产品不是聊天机器人，也不是“一键生成小说”的玩具。它是一个面向网文作者的 AI 写作生产平台：传统写作工作台负责承载作品，AI 模型负责审稿、补强、改写和结构辅助，平台画像库负责让作品从一开始就面向起点、番茄、七猫、晋江、知乎盐选、WebNovel、Royal Road、Wattpad 等真实平台目标。

产品验收判断：如果 MVP 做完后用户只能和 AI 聊天，这个项目失败；如果用户不调用 AI 也能管理一部长篇，调用 AI 后能更快审稿和改稿，这个项目才有地基。

## 2. 目标用户

### 新人作者

痛点：

1. 不知道选番茄、七猫、起点、知乎盐选还是海外平台。
2. 不知道 1 万字、5 万字、30 万字分别该怎么搭结构。
3. 容易第一章没钩子，三章后没有追读理由。

需要：

1. 平台选择向导。
2. 篇幅模板。
3. 钩子检查和章节检查。

### 稳定更新作者

痛点：

1. 长篇写到中后期人物、伏笔、设定容易乱。
2. 支线和主线脱节。
3. 每天更新时没有时间重读全部前文。

需要：

1. 大纲树。
2. 人物库。
3. 世界观库。
4. 伏笔表。
5. 连续性审稿。

### 工作室作者

痛点：

1. 多人协作时标准不统一。
2. 模型调用成本不可控。
3. 不同平台稿件需要不同审稿标准。

需要：

1. 模型配置。
2. AI 任务日志。
3. 平台适配器。
4. 结构化审稿结果。

### 出海作者

痛点：

1. 中文网文逻辑直接翻译后不一定适合海外读者。
2. WebNovel、Royal Road、Wattpad 的题材偏好不同。
3. 英文 synopsis、章节标题、标签需要重写。

需要：

1. 海外平台画像。
2. 中文大纲转英文 synopsis。
3. Royal Road 升级体系检查。
4. Wattpad 情绪和标签检查。

## 3. MVP 目标

MVP 要完成一个闭环：

1. 作者创建作品。
2. 作者选择目标平台和篇幅。
3. 作者搭大纲树。
4. 作者创建人物、设定、伏笔。
5. 作者写章节正文。
6. 作者调用 AI 审稿或改写。
7. 作者确认候选稿进入正文。
8. 作者导出作品资料。

## 4. MVP 不做

1. 不做社区。
2. 不做自动投稿。
3. 不做版权交易。
4. 不做收益预测。
5. 不做短剧分镜。
6. 不做多人实时协作。
7. 不做一键生成整本书。
8. 不做移动 App。

## 5. 核心信息架构


## 6. 功能需求

### 6.1 作品创建

用户故事：

作为作者，我要创建一个作品，并设置目标平台、目标篇幅、题材和更新目标，这样系统才能给我正确的平台建议。

字段：

1. 作品名
2. 目标平台
3. 目标篇幅
4. 目标字数
5. 题材
6. 简短卖点
7. 更新频率

目标平台选项：

1. 起点中文网
2. 番茄小说
3. 七猫
4. 晋江文学城
5. 知乎盐选
6. WebNovel
7. Royal Road
8. Wattpad

目标篇幅选项：

1. 1 万字短篇
2. 5-6 万字中篇
3. 30 万字以上长篇
4. 100 万字以上超长篇

验收标准：

1. 用户能创建作品。
2. 作品创建后进入工作台。
3. 工作台显示目标平台、目标字数、当前字数。
4. 平台策略根据目标平台自动加载。

### 6.2 作品概览

显示：

1. 当前字数
2. 章节数
3. 人物数
4. 伏笔数
5. 未回收伏笔数
6. 最近编辑章节
7. 待处理 AI 审稿问题
8. 平台策略摘要

验收标准：

1. 当前字数由章节正文自动汇总。
2. 未回收伏笔能从伏笔表计算。
3. 点击最近章节可进入编辑器。

### 6.3 大纲树

采用大树结构：

1. 根：核心卖点、主题、商业钩子。
2. 树干：主线。
3. 大枝：卷结构。
4. 小枝：支线。
5. 叶片：章节。
6. 土壤：平台规则、人物设定、世界观、读者预期。

节点类型：

1. RootIdea
2. MainPlot
3. Volume
4. BranchPlot
5. Chapter
6. PlatformRule
7. WorldRule

验收标准：

1. 用户能创建、编辑、删除大纲节点。
2. 章节节点能跳转到章节编辑器。
3. 支线节点能绑定章节。
4. 节点变更后自动保存。

### 6.4 章节编辑器

布局：

1. 左侧：章节目录。
2. 中间：正文编辑器。
3. 右侧：章节卡 + AI 助手。

章节卡字段：

1. 章节标题
2. 本章目标
3. 开头钩子
4. 冲突
5. 价值变化
6. 章末悬念
7. 出场人物
8. 涉及伏笔
9. 关联主线/支线
10. 平台适配提醒
11. 字数目标
12. 当前字数

验收标准：

1. 正文输入后自动保存。
2. 当前字数实时更新。
3. 章节能绑定人物和伏笔。
4. AI 生成内容默认进入候选区，不直接覆盖正文。

### 6.5 人物库

字段：

1. 姓名
2. 角色功能
3. 外在目标
4. 内在需求
5. 致命缺陷
6. 人物弧光起点
7. 人物弧光终点
8. 关系说明
9. 口癖和语气
10. 出场章节

验收标准：

1. 用户能创建人物。
2. 人物能绑定章节。
3. 章节编辑器能查看本章出场人物。
4. AI 审稿能读取人物资料。

### 6.6 世界观库

字段：

1. 世界规则
2. 等级体系
3. 技能体系
4. 势力结构
5. 地图地点
6. 道具物品
7. 禁忌和代价
8. 平台适配说明

验收标准：

1. 用户能创建世界观条目。
2. 条目能按类型筛选。
3. AI 审稿能读取世界观资料。

### 6.7 伏笔表

字段：

1. 伏笔名称
2. 埋设章节
3. 回收章节
4. 关联人物
5. 关联主线/支线
6. 当前状态
7. 备注

状态：

1. 未埋设
2. 已埋设
3. 待回收
4. 已回收
5. 已废弃

验收标准：

1. 用户能创建伏笔。
2. 用户能把伏笔绑定到章节。
3. 概览页显示未回收伏笔数量。
4. AI 能检查当前章节是否遗漏伏笔。

### 6.8 平台策略库

每个平台包含：

1. 平台定位
2. 适合题材
3. 推荐字数
4. 更新节奏
5. 章节节奏
6. 开头要求
7. 审稿重点
8. 风险提示

MVP 平台：

1. 起点中文网
2. 番茄小说
3. 七猫
4. 晋江文学城
5. 知乎盐选
6. WebNovel
7. Royal Road
8. Wattpad

范围收口：剩余 10 个扩展平台不再添加，不作为当前版本缺口；当前版本只把以上 8 个核心平台的写作、投稿、复盘闭环做扎实。

验收标准：

1. 用户能查看平台画像。
2. 作品创建时能选择平台。
3. 章节编辑器能显示平台提醒。
4. AI 审稿能带入平台规则。

### 6.9 模型设置

支持提供商：

1. Claude
2. DeepSeek
3. Gemini
4. GPT
5. OpenAI-compatible
6. Ollama 预留

配置字段：

1. 提供商
2. Base URL
3. API Key
4. 默认模型
5. 是否启用
6. 最大上下文

验收标准：

1. 用户能新增模型配置。
2. 用户能启用/停用配置。
3. API Key 不在前端明文展示。
4. AI 任务能选择模型。

### 6.10 AI 助手

MVP 任务：

1. 检查开头钩子
2. 检查章节爽点
3. 检查人物弧光
4. 检查章末悬念
5. 检查伏笔回收
6. 改写为番茄风
7. 改写为起点风
8. 压缩为知乎盐选短篇结构
9. 生成 WebNovel 英文 synopsis
10. 检查 Royal Road 升级体系
11. 检查 Wattpad 标签和关系张力

AI 输出类型：

1. 审稿报告
2. 改写候选
3. 大纲建议
4. 平台适配建议

验收标准：

1. AI 输出不能自动覆盖正文。
2. 改写候选必须由用户点击应用。
3. 审稿报告要包含问题、理由、建议。
4. 每次 AI 调用记录模型和 token 用量。

### 6.11 导出

导出格式：

1. Markdown
2. docx
3. 章节 zip
4. 大纲 Markdown
5. 人物设定 Markdown
6. 伏笔表 CSV

验收标准：

1. 用户能导出完整作品。
2. 用户能只导出大纲。
3. 用户能只导出人物设定和伏笔表。

## 7. 平台规则摘要

### 起点中文网

长篇、世界观、升级体系、主线持续性、IP 改编潜力。

### 番茄小说

新人友好、快节奏、强钩子、爽点密集、前三章关键。

### 七猫

稳定收益、保底导向、长篇更新、女频言情和下沉市场。

### 晋江文学城

人物关系、情感推进、言情/纯爱、审核严格。

### 知乎盐选

短篇反转、第一人称、悬疑复仇、1-15 万字。

### WebNovel

英文出海、玄幻系统、升级流、章节标题和 synopsis 英文化。

### Royal Road

LitRPG、Progression Fantasy、技能树、数值成长、硬核升级。

### Wattpad

欧美言情、狼人、豪门、青春、标签策略、关系张力。

## 8. 成功指标

MVP 产品指标：

1. 用户能在 10 分钟内创建作品并写出第一章草稿。
2. 用户能在 15 分钟内完成一个大纲树雏形。
3. 用户能在 5 分钟内配置一个模型。
4. 用户能对章节完成一次 AI 审稿。
5. 用户能导出 Markdown 文稿。

MVP 质量指标：

1. AI 输出不会覆盖用户正文。
2. 章节、人物、伏笔之间能互相绑定。
3. 平台策略能进入 AI 审稿上下文。
4. 没有目标平台的作品不能进入 AI 平台适配任务。
